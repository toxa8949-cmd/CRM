# Rower CRM

CRM для магазину: продажі (каса), склад, клієнти, витрати, звіти.
Стек: Next.js 14 + React 18 + Supabase.

## Розділи
- **Панель** — показники за сьогодні/місяць, склад, останні продажі
- **Продажі** — історія чеків, деталі, повернення (товар повертається на склад)
- **Каса** — оформлення продажу зі списанням складу
- **Склад** — товари, залишки, надходження, маржа, поріг "мало на складі"
- **Клієнти** — база з історією покупок
- **Витрати** — облік витрат за категоріями
- **Звіти** — період, топ товарів, графік по днях, чистий результат, податок 3%
- **Категорії** — категорії товарів

## Запуск

### 1. Supabase
Створіть проект на supabase.com → SQL Editor → вставте весь `sql/schema.sql` → Run.
(Схема створює таблиці, функцію `create_sale`, RLS-політики і демо-дані.)

### 2. Змінні оточення
Скопіюйте `.env.local.example` у `.env.local` і впишіть з Supabase → Settings → API:
```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

### 3. Локально
```
npm install
npm run dev
```

### 4. Деплой на Vercel
- Залийте проект у GitHub
- Vercel → Import → додайте обидві змінні оточення → Deploy

## Як працює продаж
Каса викликає серверну функцію `create_sale` (RPC). Вона атомарно: створює чек,
додає позиції, списує склад, рахує суму і прибуток. Якщо товару не вистачає —
продаж не проходить, склад не змінюється.
